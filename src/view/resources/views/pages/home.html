Monolog Framework: CLI Commands & Directives

üöÄ Getting Started

To start the application, it's recommended to use the following commands to ensure a smooth development experience. The application comes pre-configured with Tailwind CSS, and these commands will ensure everything runs correctly.

  • NPM Run Dev: Use this command to start the development server and enable live-reloading with Tailwind CSS.
  • PHP Mono.php Server: This command starts the built-in PHP server for the Monolog application. It's essential for running the application in a local environment.

üîß Available CLI Commands

Here‚Äôs a list of the available CLI commands that can be executed within the Monolog framework:

Command Description
`php mono.php server` Starts the built-in PHP server.
`php mono.php standard` Disables the application.
`php mono.php make:controller [name]` Creates a new controller.
`php mono.php make:model [name]` Creates a new model.
`php mono.php make:migration [name]` Creates a new migration file.
`php mono.php make:middleware [name]` Creates a new middleware file.
`php mono.php make:seeder [name]` Creates a new seeder.
`php mono.php make:view [name]` Creates a new view file.
`php mono.php make:layout [name]` Creates a new layout template.
`php mono.php make:component [name]` Creates a new component file.
`php mono.php make:mail [name]` Creates a new mail layout file template.
`php mono.php make:seed` Executes all seeder classes to insert initial data into the database.
`php mono.php make:migrate` Executes all pending migrations in the database.

üîß Directory Structure After Running Commands

After executing the CLI commands, the generated files will be stored in the following directories:

  • /src: Contains the main source files for the framework.
    • /app: Core application files.
      • /config: Configuration files (e.g., includes.php, stacking.php).
      • /console/commands: Contains custom CLI commands.
        • Files such as MakeComponent.php, MakeController.php, MakeModel.php, MakeMigration.php, among others.
      • /controller: Contains controller files.
      • /helpers: Helper functions for various purposes.
      • /http: HTTP-related files like Request.php, Response.php, Router.php.
      • /middlewares: Middleware files for API and Web.
      • /resources: Resource files like views and components.
      • /database: Database-related files, including migrations and seeders.
      • /model: Contains model files.
      • /router: Routing files (e.g., web.php).
      • /view: Contains view files.
        • /public: Public assets like CSS, images, and JS.
        • /resources: Additional resource files, including views.
    • /vendor: Composer dependencies.
    • /node_modules: Contains NPM packages, including Tailwind CSS.

‚öôÔ∏è Monolog Directives

Monolog comes with several custom directives that make development faster and cleaner. Here‚Äôs an explanation of some common ones:

  • [@component]: Used to include a Blade component. It helps in reusing UI elements across your application.
  • [@assets]: Used to reference static assets like CSS, JS, and image files, automatically linking them to the public directory.
  • [@endif]: Complements if statements for logic separation in Blade files.
  • [@forelse]: Similar to the regular @foreach directive, but adds logic to handle empty arrays with a custom message.
